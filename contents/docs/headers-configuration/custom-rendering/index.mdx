## Custom Rendering

Override default rendering with custom components for headers and cells.

### Custom Cell Renderer

Use `customRender` to completely control cell output:

```tsx
{
  id: "status",
  text: "Status",
  customRender: (item, header, fromArrayData, isDisabled) => {
    const status = item[header.id]
    const colors = {
      Active: "bg-green-100 text-green-800",
      Pending: "bg-yellow-100 text-yellow-800",
      Inactive: "bg-red-100 text-red-800",
    }

    return (
      <span className={`px-2 py-1 rounded ${colors[status]}`}>
        {status}
      </span>
    )
  },
}
```

### Custom Cell Renderer Parameters

| Parameter       | Type                | Description                      |
| --------------- | ------------------- | -------------------------------- |
| `item`          | `TableItem`         | Current row data                 |
| `header`        | `BaseTableHeader`   | Column configuration             |
| `fromArrayData` | `{ index: number }` | Index when rendering array items |
| `isDisabled`    | `boolean`           | Whether the cell is disabled     |

### Custom Header Renderer

Use `customHeader` to customize column headers:

```tsx
{
  id: "priority",
  text: "Priority",
  customHeader: (header) => (

      <span>{header.text}</span>
    </div>
  ),
}
```

### Combining with Edit Options

Custom rendering works alongside edit options. The custom render is displayed, but clicking enables the editor:

```tsx
{
  id: "price",
  text: "Price",
  editOptions: {
    editable: true,
    type: "number",
  },
  customRender: (item, header) => (
    <span className="font-mono">
      ${item[header.id].toFixed(2)}
    </span>
  ),
}
```
