## Filtering

Enable column filtering to let users narrow down data.

### Enable Filters

Add `hasFilter: true` to your headers:

```tsx
const headers: BaseTableHeader[] = [
  {
    id: "status",
    text: "Status",
    hasFilter: true,
  },
  {
    id: "department",
    text: "Department",
    hasFilter: true,
  },
]
```

### Active Filters

Pass the current filter state with `activeFilters`. If you don't provide it, users will still be able to filter items. This is just to initialize filters when needed. Here is the interface for `activeFilters`:

```ts
interface ActiveTableFilter {
  headerId: string
  itemsToHide: string[] | number[]
  numberConditions?: NumberCondition[]
}

type Operator = "=" | ">" | "<" | ">=" | "<=" | "between"

interface NumberCondition {
  id: string
  operator: Operator
  value1: number
  value2?: number // Required when operator is "between"
}
```

Basic string filtering:

```tsx
<BaseTable
  headers={headers}
  items={items}
  activeFilters={[
    {
      headerId: "status",
      itemsToHide: ["Inactive"],
    },
    {
      headerId: "department",
      itemsToHide: ["HR"],
    },
  ]}
/>
```

Number filtering with conditions:

```tsx
<BaseTable
  headers={headers}
  items={items}
  activeFilters={[
    {
      headerId: "salary",
      itemsToHide: [],
      numberConditions: [
        {
          id: "salary-min",
          operator: ">=",
          value1: 50000,
        },
      ],
    },
    {
      headerId: "age",
      itemsToHide: [],
      numberConditions: [
        {
          id: "age-range",
          operator: "between",
          value1: 25,
          value2: 65,
        },
      ],
    },
  ]}
/>
```

### Filter with List Columns

Combine filtering with list-type columns for dropdown filters:

```tsx
{
  id: "status",
  text: "Status",
  hasFilter: true,
  editOptions: {
    editable: true,
    type: "list",
    options: ["Active", "Pending", "Inactive"],
  },
}
```
