## Editing Overview

The editing system allows users to modify data directly in the table. To make a column editable, add the optional `editOptions` field to your table headers. You must also provide the appropriate callback function to handle and persist updates to your data.

### A Simple Example

```tsx
// 1. Define editable columns
const headers: BaseTableHeader[] = [
  {
    id: "name",
    text: "Name",
    editOptions: {
      editable: true,
      type: "string",
    },
  },
]

// 2. Handle changes
<BaseTable
  headers={headers}
  items={items}
  onChange={(updatedItem, originalIndex) => {
    // 3. Validate and persist
    updateItem(updatedItem)
  }}
/>
```

### Edit Options in Detail

Configure editing per column:

```ts
editOptions: {
  editable?: boolean;
  isDisabled?: (
    item: TableItem,
    fromArrayData?: { index: number }
  ) => boolean;
  notApplicable?: (
    item: TableItem,
    fromArrayData?: { index: number }
  ) => boolean;
  greyedOutIfNotEditable?: boolean;
  required?: boolean;
  type: TableHeaderType;
  options?: string[]; // only valid for List type
  defaultValue?: string | number;
  canAddNewOption?: boolean; // only valid for List type
}
```

### Editor Types

- **String** - Text input editor
- **Number** - Numeric input editor
- **List** - Dropdown/select editor

### Change Events

The table triggers different callbacks based on the scope:

| Callback            | Use Case                     |
| ------------------- | ---------------------------- |
| `onChange`          | Single cell updated          |
| `onBulkChange`      | Multiple cells updated       |
| `onBulkFieldChange` | Multiple rows in same column |

### Validation

Control when cells can be edited:

```tsx
isDisabled: (item, fromArrayData) => item.locked === true
notApplicable: (item) => item.type === "service"
```
